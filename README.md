# Mini-IT TT Sergey Andreev

Проект содержит базовый функционал игр: Match3, Merge, Arkanoid.

Архитектурные принципы и почему так

  * Разделение ответственности (SRP): менеджер грида только хранит и оперирует клетками; логика матчинга — в отдельном сервисе MatchFinder.

  * Интерфейсы (IGridItem, IPoolable) — демонстрация полиморфизма и лежит в основе переиспользования.

  * События (C# events) для коммуникации (например, OnScoreChanged, OnLevelComplete)

  * Generics: GridCore<TCell> и ObjectPool<T> — демонстрация шаблонного программирования.

  * LINQ удобно и выразительно для фильтраций (напр. поиск соседей, групп).

  * ScriptableObjects — хранение конфига отдельно от кода.

  * Object Pooling — для игровых объектов (создание/уничтожение нежелательно во время игры).

  * Код оформлен в стиле, близком к вашим правилам: явные модификаторы, Pascal для публичных свойств, camel для локалов, XML комментарии, пробелы/пустые строки.
